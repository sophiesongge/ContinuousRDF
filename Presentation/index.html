Apache Storm is a distributed computation framework

Apache storm

  • Spout
  • Bolt
  • Topology
A Storm application is designed as a "topology" in the shape of a directed acyclic graph (DAG) with spouts and bolts acting as the graph vertices. Edges on the graph are named streams and direct data from one node to another. Together, the topology acts as a data transformation pipeline.

Query Decomposition

  • 1-Variable Join
  • 2-Variable Join
  • Multi-Variable Join
We splited queries into 3, graph based categories.

1-Variable Join

find the subject for which p1 is equal to o1, p2 is equal to o2 and p3 is equal to o3.

1-Variable Join

SELECT author WHERE
Paper = kNN AND
Work = INRIA AND
diplome = Ph.D
Find the authors for paper kNN who works in INRIA and who has a Ph.D diplome

2-Variable Join

find the subject and p3 for which p1 is equal to o1, p2 is equal to o2

2-Variable Join

SELECT author, diplome WHERE
Paper = kNN AND
Work = ECP

Multi-Variable Join

find the subject,p2,p3 for which p1 is equal to o1.

Multi-Variable Join

SELECT author, work, diplome WHERE
Paper = kNN
to find the authors for paper kNN, and the place they work, and their diplome

Graph to Topology

1-Variable Join Topology

SELECT author WHERE Paper = kNN AND work = ECP AND diplome = Ph.D
find the authors for paper kNN who works in ECP and who has a Ph.D diplome. we can choose any predicate for builder and prober bolts.

2-Variable Join Topology

SELECT author,diplome WHERE Paper = kNN AND work = ECP
find the authors for paper kNN and their diplome who works in ECP we can't' choose any predicate for builder and prober bolts. We have to select predicate diplome as our prober.

Multi-Variable Join Topology

SELECT authors,work,diplome WHERE Paper = kNN
find the authors for paper kNN, and the place they work, and their diplome we can't' choose any predicate for builder and prober bolts. We have to select predicate diplome and work as our prober and paper as builder.

Sliding Window Integration

Without Sliding Window

Bolt builder sends one bloom filter at a time to the bolt prober, prober search data from that bloom filter. we want to add more flexibility in searching interms of time and rdf triplse. We can achive this by increasing the bloom filter size, but if we increase the bloom filter size it means we will increase the amount of data transfer and also time delay.

Sliding Window Integration

Sliding window size = #generations x generation size

Without increasing bloom filter size we can acieve this by adding sliding window at the prober. So in this way, the amount of data transfer will be same and no delay.
